以下の要件でFastAPI + SQLAlchemy + Reactでプロジェクトをセットアップしてください。

【前提】
- ユーザー認証は現時点では不要です。ユーザー区別なしでシンプルに実装してください。
- フロントエンドはReact + Vite + Tailwind CSS構成とし、APIはすべてバックエンド側FastAPIで処理します。

【必要な機能】
① PDFアップロード → QA CSV生成機能
- PDFファイルをアップロードし、サーバー側で保存
- バックエンド側でQA処理（今回はダミー処理で良い）
- 生成されたCSVファイルをダウンロードできるAPI

② プロンプト編集・保存機能
- プロンプト一覧取得（現状は1件のみ）
- プロンプト内容編集
- 編集した内容を保存するAPI（PUT）

【バックエンド設計】
- ORMはSQLAlchemyを使用
- DBテーブルは prompts と uploaded_files の2種類
- Alembicは使わなくてもOK
- JWTや認証は後回し

【フロントエンド設計】
- Reactコンポーネントは最低限
- PDFアップロードボタンと状態表示
- プロンプト編集フォームと保存ボタン
- axiosでAPI連携

備考

AITalk

◎ 日本語特化。イントネーション・発音精度高い。感情表現可（一部話者）

◎ オンプレ対応。低遅延でリアルタイム応答可

◎ 法人契約で可能

○ REST API / SDK

駅・車内放送など実績多数。オンプレ設置も可

Amazon Polly

○ ニューラル音声あり。SSMLで制御可能

◎ ストリーミング対応で高速

◎ 従量課金制

◎ AWS SDK / Genesys対応

Genesys Cloudとの親和性が高い

Cartesia.ai

◎ 圧倒的自然さ・感情調整可能

◎ 最速クラス（90ms以下）

◎ 要商談

○ REST API

Sonicモデルが特徴。法人導入実績あり

CoeFont

◎ 声種が豊富。自然さ高いが品質は声により差あり

○ API応答は標準的

◎ Plusプラン以上

○ REST API

クローン声やタレント声も利用可。要個別契約の場合あり

Google TTS

○ WaveNet音声。自然だが感情表現は少なめ

○ ストリーミングあり（Chirp3等）

◎ 従量課金制

◎ SDK豊富 / Genesys統合可

2025年以降の公式連携は終了予定（代替手段あり）

VoiceText（HOYA）

◎ 日本語に強く、感情音声あり（怒り・喜びなど）

◎ オンプレ利用可

◎ 法人ライセンス

○ SDK / サーバ製品

IVRやナビ用途の実績多数。契約は要問合せ

考察・選定ポイント

自然さ重視の場合、Cartesia.ai や AITalk が有力。特に Cartesia は感情表現の豊かさが突出。

リアルタイム応答重視なら Cartesia.ai、AITalk、Polly が安定。

クラウド／オンプレの柔軟性を求める場合、AITalk や VoiceText が適す。

コスト管理・拡張性重視なら Amazon Polly や Google TTS が導入しやすい。
